🧠 Co je Git a proč ho chceme?

Představte si, že programujete robota, který jezdí po čáře. Máte kód, který funguje. Pak zkusíte přidat zatáčení — a najednou nefunguje nic. Ctrl+Z nepomáhá, protože jste mezitím uložili 10×.

Přesně tohle řeší Git — je to systém, který si pamatuje každou verzi vašeho kódu. Jako nekonečné „undo" tlačítko s popiskem, co jste kdy změnili.

📁 Bez Gitu

  • robot_v1.py
  • robot_v2_funguje.py
  • robot_FINAL.py
  • robot_FINAL_opravdu.py
  • robot_backup_stary.py

✨ S Gitem

  • Jeden soubor: main.py
  • Celá historie změn
  • Kdo co změnil a proč
  • Návrat k libovolné verzi
  • Bezpečné experimentování

Git vs. GitHub

🔧 Git

  • Program na vašem počítači
  • Ukládá historii změn
  • Funguje i offline
  • Zdarma a open-source

🌐 GitHub

  • Web, kam nahrajete kód
  • Sdílení s celým týmem
  • Záloha v cloudu
  • Zdarma pro studenty
💡 Analogie: Git je jako deník, kam si zapisujete, co jste na robotu změnili. GitHub je jako nástěnka ve třídě, kam ten deník pověsíte, aby si ho přečetli i ostatní.

⚙️ Instalace a nastavení

1
Nainstalujte Git

Stáhněte z git-scm.com a nainstalujte (výchozí nastavení stačí).

# Ověření instalace (v terminálu VS Code)
$ git --version
git version 2.44.0
2
Nastavte si jméno a email

Git potřebuje vědět, kdo změny dělá:

$ git config --global user.name "Pepa Novák"
$ git config --global user.email "pepa@email.cz"
3
Vytvořte si účet na GitHubu

Jděte na github.com → Sign up. Zdarma!

4
VS Code rozšíření

VS Code má Git podporu vestavěnou! Stačí otevřít panel Source Control (ikona větvičky vlevo).

💡 Tip: Ve VS Code otevřete terminál přes Ctrl + ` (backtick) — všechny git příkazy zadáváte sem.

🌐 GitHub – sdílení kódu

1
Vytvořte repozitář na GitHubu

Na github.com klikněte New repository → pojmenujte ho (např. lego-robot) → Create.

2
Propojte lokální repozitář 
$ git remote add origin https://github.com/vas-ucet/lego-robot.git
3
Nahrajte kód (push) 
$ git push -u origin main

Poprvé se Git zeptá na přihlášení do GitHubu.

4
Stáhnout změny od ostatních (pull) 
$ git pull

Pokud kamarád něco změnil na GitHubu, tímto si stáhnete jeho změny.

5
Naklonovat existující projekt 
$ git clone https://github.com/vas-ucet/lego-robot.git

Nový člen týmu si takto stáhne celý projekt i s historií.

🤖 Příklad: Projekt LEGO robota

Pojďme projít reálný příklad – od nuly k funkčnímu projektu s Pybricks.

Struktura projektu

📁 lego-robot/
📄 .gitignore
📄 README.md
🐍 main.py

Krok 1 – Založení projektu

$ mkdir lego-robot
$ cd lego-robot
$ git init

Krok 2 – Soubor .gitignore

Řekneme Gitu, které soubory NEMÁ sledovat:

# .gitignore
__pycache__/
*.pyc
.vscode/
.pybricks/

Krok 3 – První kód robota

# main.py – Základní jízda
from pybricks.hubs import PrimeHub
from pybricks.pupdevices import Motor
from pybricks.parameters import Port, Direction
from pybricks.tools import wait

hub = PrimeHub()

# Motory
levy_motor = Motor(Port.A)
pravy_motor = Motor(Port.B, Direction.COUNTERCLOCKWISE)

# Jeď dopředu 2 sekundy
levy_motor.run(500)
pravy_motor.run(500)
wait(2000)

# Zastav
levy_motor.stop()
pravy_motor.stop()

hub.speaker.beep()

Krok 4 – První commit

$ git add .
$ git commit -m "Základní jízda dopředu na 2 sekundy"

Krok 5 – Přidáme sledování čáry

# main.py – Přidáme senzor a PID
from pybricks.hubs import PrimeHub
from pybricks.pupdevices import Motor, ColorSensor
from pybricks.parameters import Port, Direction
from pybricks.tools import wait

hub = PrimeHub()

levy_motor = Motor(Port.A)
pravy_motor = Motor(Port.B, Direction.COUNTERCLOCKWISE)
senzor = ColorSensor(Port.C)

# PID konstanty
ZAKLADNI_RYCHLOST = 300
KP = 1.2

# Sledování čáry
while True:
    odrazivost = senzor.reflection()
    chyba = odrazivost - 50  # 50 = ideální hodnota
    korekce = KP * chyba

    levy_motor.run(ZAKLADNI_RYCHLOST + korekce)
    pravy_motor.run(ZAKLADNI_RYCHLOST - korekce)

    wait(10)

Krok 6 – Commit a push

$ git add main.py
$ git status
Changes to be committed:
  modified:   main.py

$ git commit -m "Přidáno sledování čáry s P regulátorem"
$ git push

Krok 7 – Uups, nefunguje! Návrat zpět

# Podívám se do historie
$ git log --oneline
c4a8f2b Přidáno sledování čáry s P regulátorem
a1b3d5e Základní jízda dopředu na 2 sekundy

# Vrátím se ke staré verzi
$ git checkout a1b3d5e

# Funguje! Vrátím se zpět a opravím novou verzi
$ git checkout main
💡 Reálný scénář: Na soutěži potřebujete rychle přepnout mezi programem pro jízdu po čáře a programem pro bludiště. S Gitem to zvládnete jedním příkazem!

📋 Tahák – nejdůležitější příkazy

🏁 Začátek
$ git init                   # Nový repozitář
$ git clone <url>             # Stáhnout existující
💾 Ukládání změn
$ git status                  # Co se změnilo?
$ git add .                   # Připravit vše
$ git commit -m "popis"       # Uložit do historie
🌐 Sdílení (GitHub)
$ git push                    # Nahrát na GitHub
$ git pull                    # Stáhnout z GitHubu
🔍 Prohlížení
$ git log --oneline           # Historie
$ git diff                    # Co jsem změnil?
⏪ Záchrana
$ git checkout <hash>         # Návrat k verzi
$ git checkout main           # Zpět na nejnovější
🔄 Denní workflow
# Ráno (stáhni novinky)
$ git pull

# Programuj... testuj... funguje!

# Ulož práci
$ git add .
$ git commit -m "Vylepšen PID regulátor"
$ git push
💡 Zlaté pravidlo: git status → git add . → git commit -m "..." → git push. Tohle je 90 % práce s Gitem!
